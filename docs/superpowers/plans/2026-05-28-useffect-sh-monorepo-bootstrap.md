# useffect.sh Monorepo Bootstrap Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Scaffold a Turborepo + Bun monorepo whose single `apps/web` (Next.js 15) auto-deploys to `useffect.sh` via Vercel on push to `main`.

**Architecture:** Bun workspaces (`apps/*`, `packages/*`) orchestrated by Turborepo. One Next.js 15 App Router web app with Tailwind v4, Biome, Vitest. CI gate on GitHub Actions. Deploy via Vercel dashboard with Root Directory = `apps/web` (no `vercel.json`).

**Tech Stack:** Bun 1.3.x · Turborepo 2.x · Next.js 15 · TypeScript strict · Tailwind v4 · Biome 2.x · Vitest + Testing Library + jsdom · Husky + lint-staged.

**Spec:** `docs/superpowers/specs/2026-05-28-useffect-sh-monorepo-bootstrap-design.md`

---

## CRITICAL EXECUTION CONSTRAINTS

- **NEVER run `git add`, `git commit`, `git push`, or any state-changing git command.** The user reviews and commits everything. Tasks end in verification commands, not commits. The final handoff section gives the user the commit command to run themselves.
- **Bun only.** Never invoke `npm`, `pnpm`, or `yarn`. Use `bun install`, `bun add`, `bun run`, `bun x`.
- **Pin nothing by hand.** Let `bun add` resolve latest stable; `bun.lock` is the lockfile of record.
- **Strict TS, no `any`, no `@ts-ignore`.** Tailwind classes only — no inline styles, no CSS modules.
- This is a scaffolding plan: most "tests" are verification commands. Genuine TDD applies only to the homepage smoke test (Task 7).

```yaml
name: CI

  push:

    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: oven-sh/setup-bun@v2
        with:
          bun-version: latest

      - uses: actions/setup-node@v4
        with:
          node-version: 22

      - name: Cache Turbo
        uses: actions/cache@v4
        with:
          path: .turbo
          key: ${{ runner.os }}-turbo-${{ hashFiles('bun.lock') }}
          restore-keys: |
            ${{ runner.os }}-turbo-

      - name: Install dependencies
        run: bun install --frozen-lockfile

      - name: Lint
        run: bun run lint

      - name: Typecheck
        run: bun run typecheck

      - name: Test
        run: bun run test:ci

      - name: Build
        run: bun run build
```

## Handoff to the user (you run all git + Vercel steps)

Once every task verifies, **the user** performs these — Claude does none of them:

### 1. Commit & push

```bash
git add .
git commit -m "feat: bootstrap monorepo with web app and CI"
git push origin main
```

Review the diff first (`git status`, `git diff --staged`). The `docs/superpowers/` spec and plan can be included in the same commit or a separate one — user's choice.

### 2. Vercel dashboard

- Import the repo `magrinj/useffect.sh`.
- **Root Directory → `apps/web`** (this is the key setting; Vercel auto-detects Next.js + Turborepo and installs from the repo root).
- Framework preset: Next.js (auto-detected). Build/Install commands: leave as Vercel defaults — do **not** add overrides; there is no `vercel.json` by design.
- Add domains: `useffect.sh` and `www.useffect.sh`.

### 3. DNS at registrar

Set the records Vercel displays for your domain. Vercel's standard values are:
- `A` record on `@` → `76.76.21.21`
- `CNAME` on `www` → `cname.vercel-dns.com`

Confirm against the exact values shown in the Vercel Domains tab (they can be account-specific). Wait 5–30 min for propagation.

### 4. Result

`useffect.sh` serves the `// mounting...` teaser. Every push to `main` redeploys; every PR gets a preview URL.
